Creating Multipage React Application Using React Router DOM

Step-by-step process:

1. **Install React Router DOM:**
	- Ran: `npm install react-router-dom`
	- This added the official routing library for React to the project.

2. **Set Up Router in App:**
	- Imported `BrowserRouter`, `Routes`, and `Route` from `react-router-dom` in `src/App.jsx` (or `main.jsx`).
	- Wrapped the main app component with `<BrowserRouter>` to enable routing.

3. **Create Page Components:**
	- Created separate React components for each page (e.g., `Home.jsx`, `About.jsx`, `Contact.jsx`, etc.) in the `src/pages` directory.

4. **Define Routes:**
	- Used the `<Routes>` and `<Route>` components to map URLs to page components.
	- Example:
	  ```jsx
	  <Routes>
		 <Route path="/" element={<Home />} />
		 <Route path="/about" element={<About />} />
		 <Route path="/contact" element={<Contact />} />
	  </Routes>
	  ```

5. **Add Navigation:**
	- Used `<Link>` or `<NavLink>` from `react-router-dom` to create navigation links between pages.
	- Example:
	  ```jsx
	  <nav>
		 <Link to="/">Home</Link>
		 <Link to="/about">About</Link>
		 <Link to="/contact">Contact</Link>
	  </nav>
	  ```

6. **Test Multipage Navigation:**
	- Ran the development server with `npm run dev`.
	- Navigated between pages using the links and confirmed the URL and content updated without a full page reload.

7. **Optional Enhancements:**
	- Used nested routes for layouts or subpages.
	- Added 404 Not Found route for unmatched URLs.
	- Used route parameters for dynamic pages (e.g., `/product/:id`).

---

This process enables a true multipage experience in a single-page React app, using React Router DOM for fast, client-side navigation.
